# Security Policy

## Supported Versions

Security updates are currently provided for the latest active version of OryCMS.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues or discussions.

Instead, report them privately with:

- A description of the issue
- Steps to reproduce, if available
- Potential impact
- Any suggested remediation

If a private security contact is added later, this file should be updated with that channel.

Until then, coordinate through the repository maintainers and avoid public disclosure until the issue has been reviewed and addressed.

## Response Process

When a valid report is received, maintainers will aim to:

- Confirm receipt
- Assess impact and severity
- Work on a fix
- Coordinate responsible disclosure
