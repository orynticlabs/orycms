# Contributing to OryCMS

Thanks for your interest in contributing to OryCMS.

## Getting Started

1. Fork the repository.
2. Create a feature branch for your work.
3. Make focused, well-scoped changes.
4. Test your changes locally.
5. Open a pull request with a clear description.

## Development Guidelines

- Keep pull requests small and easy to review.
- Follow the existing code style and project structure.
- Update documentation when behavior or setup changes.
- Link related issues when possible.

## Pull Request Expectations

Before opening a pull request, please make sure:

- The change is tested.
- The code builds and runs locally.
- Documentation is updated if needed.
- The PR explains what changed and why.

## Reporting Bugs

Please use the GitHub issue templates when reporting bugs. Include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details
- Logs or screenshots when relevant

## Suggesting Features

Feature requests are welcome through GitHub Issues or Discussions. Clearly describe the problem you want to solve and the proposed improvement.
